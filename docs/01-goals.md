# 01 · 目标与一期定义

## 1. 最终目标（North Star）

**让人类专家成为 agent 网络的一等公民节点：任何 AI 系统都能像调用一个模型、或一个 agent 一样，调用一位人。**

两种调用形态，对应两期：

| | 一期 · 同步模式 | 二期 · 异步模式 |
|---|---|---|
| 人扮演 | **模型**（接管 harness 的大脑） | **Agent**（独立接单交付） |
| 接口 | Anthropic Messages + OpenAI Chat Completions；Responses API 由 P1-M0 实测裁决 | A2A + MCP |
| 工作形态 | 人守着任务，实时指挥需求方机器 | 接单后自主深度工作，diff 交付 |
| 典型场景 | 远程调试（adb）、排障、结对、答疑 | 功能开发、重构、传统外包 |
| 状态 | **Pre-P1-M0：设计完成，待兼容性实验** | 设计归档 [phase2-async-mode.md](phase2-async-mode.md)；协议核心经有限状态模型检查，尚未实现 |

远景：专家网络与市场；需求方**单独 opt-in** 后，completion 往返可沉淀为人类专家轨迹数据。

## 2. 一期决策记录：为什么"人当模型"

需求推演中反复出现同一模式——每当为"人=Agent"路线补一个能力，结论都是"人=模型"形态免费拥有它：

| 能力 | 人=Agent 的代价 | 人=模型 |
|---|---|---|
| 文件就地变化 | patch 传输 + 五级 apply + baseline 对齐 | 免费（harness 的 edit 在需求方机器执行） |
| 远程命令（adb） | 专用子协议 + 授权模型 | 免费（harness 的 bash 工具） |
| 执行授权 | 自建 permission 关卡 | 免费（harness 自带 allowlist/确认） |
| Esc 中断 / Esc Esc 回滚 | 消息分级 + 三步回滚 + 服务端事务 | 复用 harness checkpoint；gateway 仍管理跨回合任务状态 |
| 两侧一致性 | 五条 TLA+ 不变量 + 运行时校验 | 从 git/patch 全局问题**缩小为逐 tool-call 确认**（唯一真相=需求方工作树，失败显式化）——不是消失、是变小 |

加三个工程理由：工程量小一个数量级（无 git 引擎；但仍需跨回合任务状态，非完全无状态）；**Chat 档**接入 = 改一行 base_url（Remote tools / Workspace 档需更多，见 [02](02-gateway.md) §1）；最快验证核心产品假设。

代价（如实）：人需**同步在场**（守着任务连续响应）——因此深度异步工作（外包大单）留给二期；上下文与工具面由对方 harness 支配；assistant 消息不能回传图片（协议限制）。

## 3. 角色与体验之锚

- **需求方**：任何支持自定义模型端点的 harness 的用户（或无人值守系统）。体验之锚：**与用一个很慢但很聪明的模型无差别**——工具照常执行、界面照常流式、错误照常重试。
- **接单人**：TUI 前的人类专家，**在自己的 IDE 里干活**（Claude Code/cursor/手写皆可）。体验之锚：**一个沟通与交付的信使台**——读对方递来的需求、聊清楚，去自己 IDE 干活，回来核对改动、放行交付。中间几十个 tool-call 回合不暴露给人；人面对任务级节奏（聊需求→干活→交付），不逐回合扮演模型。沟通就是对话（对方发起、人回复、可反问澄清），无主动发起（[03](03-tui.md) §1–2）。

三条刚性约束：

1. **人的响应以分钟计，研发型任务可长挂数小时** → SSE 心跳保活贯穿始终，且进度可流式先吐（把长挂黑箱变成"模型逐步输出"）。200 前的机器准入失败用真实 `503/529`；200 后的人工拒单/超时只能用流内错误或断流，两条路径均须实测客户端行为。**"一个 completion 请求能挂多久"是整个方向的生死门，P1-M0 必须先验（[04](04-milestones.md)）。**
2. **人不像模型那么听话** → 一切离开本机的内容（文本、文件改动、命令）发送前人眼核对；系统做翻译与暂存，不代替确认。
3. **接单人是"不受信任的 tool_call 来源"**（与模型同构，但为思考型对手）→ 网关做**词法纵深防御**（路径规整/越界拒转）；但真正的执行闸在需求方 harness 权限系统 + caller shim（realpath/symlink/执行限制），不在网关。命令能力默认关闭、按 task 开（见 [02](02-gateway.md) §11）。

## 3.1 人不手搓工具 · 文件隐式 · 可转包

三条定义一期交互模型的决定（细节见 [03](03-tui.md) §1–2、[02](02-gateway.md) §11）：

- **人不构造 tool_call**：协议底层的"文本 + tool_calls"是管道，不暴露。人只做聊天、命令、todos；**文件是隐式的**——人在真实工作目录里用惯用工具改，fsnotify 监听 diff 自动构造文件 tool_call。
- **工具栈不暴露**：系统靠**显式 harness adapter**（版本化）声明文件/命令工具的真实语义并映射（schema 启发式只提示、不自动启用写入/执行）；对方的自定义工具是其 agent 自己的事，接单人不碰。未识别 harness 降级纯聊天。
- **可转包本机 agent**：因系统只监听工作目录、不问谁在填，接单人可把本机 Claude/Codex 指向镜像目录代劳，改动自动同步。前提：人是**监督闸非透传**（对每次交付负责）；价值**有条件**（须带来对方给不了的上下文/工具/判断，否则等于让对方绕道调普通模型）；**数据出境须披露**（对方代码经接单人流向其模型商）。

## 4. 一期场景

| # | 场景 | 说明 |
|---|---|---|
| S1 | 远程调试/排障 | 设备与环境在需求方侧（adb、内网库、"只在你环境复现"）——同步模式的杀手场景，人实时看输出连续决策 |
| S2 | 编码任务 | read → edit → bash（测试）循环，人指挥 harness 完成 |
| S3 | 咨询问答 | 纯文本回合，无工具调用 |
| S4 | 结对/评审 | 人阅读 harness 递来的代码上下文，给意见 |
| S5 | 多需求方排队 | 跨会话请求进队列，人逐个处理（同一会话内 harness 天然串行）——排队体验与模型一致 |
| S6 | 转包本机 agent | 接单人把 Claude/Codex 指向镜像目录代劳，人监督核对后交付（§3.1）——人机协作，非纯手工 |
| S7 | 异常 | 准入时无人在线 → overloaded 语义重试；流式中拒单/超时 → 流内失败；对方 Esc/断连 → TUI 中断提示；历史指纹断裂只新建 UI 分组（payload 是否保留取决于 opt-in） |

## 6. 非目标（一期）

- A2A / git 引擎 / patch 交付 / worktree / rewind 协议——全部二期（[phase2-async-mode.md](phase2-async-mode.md) 设计完备）
- 人回传图片给对方（assistant 消息协议限制；降级：文字描述或 gateway 托管 URL 进 backlog）
- 上下文管理（压缩、截断是 harness 的职责，我们只做管道）
- 多接单人路由、市场、计费
- PTY / 长驻交互进程；终端实时性——命令回合制（结果下一回合返回），实时流式改动列 backlog
- 全仓预同步（镜像目录惰性按需填充即可）；转包的技术性阻断/数据标注（一期靠核对闸 + 声明治理，见 [02](02-gateway.md) §11）
- gateway 侧 LLM 起草辅助（我们代写草稿）——注意与"接单人自带本机 agent 干活"（§3.1，一期默认支持）不同，后者是接单人自己的 agent、我们只监听工作目录
- **主动发起/反向消息**：completion 协议无回调通道，人只能响应对方请求，主动信息搭在下次响应里（"tips 过去"）；真正的反向发起（webhook）是二期 A2A 能力
- **全量 payload 长存 / 训练默认开**：默认安全——元数据审计默认；payload 默认关闭，显式开启后默认 TTL=7 天；训练与转包各自独立 opt-in（G-08、[02](02-gateway.md) §9）

## 7. 术语

| 术语 | 定义 |
|---|---|
| 方言（dialect） | Anthropic Messages、OpenAI Chat Completions 或待 P1-M0 裁决的 Responses API 协议格式；进哪种出哪种 |
| 回合请求 | 一次 completion 请求（含全量历史与工具清单），人给出一次响应 |
| workspace_key | 路径根 R / 能力档 / baseline / 镜像目录名的载体（**正确性绑它**，[05](05-p1-m0-contract.md) §1） |
| task_id | 一次工具循环 / lease / 幂等的载体（**正确性绑它**） |
| ui_conversation_group | 仅 TUI 显示分组，历史指纹启发式，可错、不影响正确性 |
| 挂起（pending） | 请求已到达、心跳保活中、等人响应的状态 |
| canonical | 方言无关的内部规范格式（messages/blocks/tools） |
| 镜像目录 | 对方工作区在接单人机器上的本地副本，人/agent 在此干活，改动经监听自动回传 |
| 基线 commit | 镜像 checkout 的起点，与对方工作树对齐一次；仅用于上下文，非传输 |
| 单一写者 | compat 下对方工作树主要由接单人 tool_call 驱动；但一致靠**逐 tool-call 确认**（成功 result 才推进 baseline），非自动对齐 |
